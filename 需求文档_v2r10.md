## 最新修改（v2r10）

### 1. 修复抓取数据导入bug（2026-04-20）
**问题描述：**
- 通过项目管理点击项目进入任务管理页面
- 点击"打开搜索"，在电商平台使用书签抓取数据
- 回到监控工具后，显示"收到X个商品数据"但导入失败

**原因分析：**
- showTaskManagement()函数会重置currentProjectId
- 导致importScrapedData()找不到目标项目

**解决方案：**
- showTaskManagement()通过项目点击进入时保留currentProjectId
- applyFilters()优先使用currentProjectId进行筛选
- updateProjectList()只在筛选公司时才过滤项目列表

**修改内容：**
```javascript
function showTaskManagement() {
    if (!currentProjectId) {
        entryMode = 'navigation';
        currentProjectId = null;
    }
    // ... 其他代码，不重置currentProjectId
}

function applyFilters() {
    var targetProjectId = currentProjectId;
    if (!targetProjectId) {
        targetProjectId = document.getElementById('filterProject').value;
    }
    // 使用targetProjectId筛选数据
}
```

## 页面结构

### 项目管理页面
```
第1行：标题
第2行：筛选公司 + 确认 + 重置 | 筛选国家 + 确认 + 重置
第3行：筛选平台 + 确认 + 重置 | 侵权线索数量排序 + 确认 + 重置
第4行：销售总额排序 + 确认 + 重置
第5行：项目列表表格（点击项目行跳转到任务管理）
```

### 任务管理页面
```
第1行：拖动书签栏
第2行：标题 + 已选数量
第3行：筛选公司 + 确认 + 重置 | 筛选项目 + 确认 + 重置
第4行：关键词搜索 + 搜索 + 重置
第5行：4个操作按钮 + 平台选择 + 确认 + 重置
第6行：侵权线索表格
```

## 已知限制

### 数据存储
- 数据存储在浏览器localStorage
- 不同用户之间数据不共享
- 清除浏览器缓存会丢失数据
- 建议定期使用"导出备份"功能

### 书签抓取
- 需要手动拖动书签到书签栏
- 抓取后需点击"导入抓取数据"按钮
- 支持平台：淘宝、天猫、京东、拼多多、1688

## 使用说明

### 个人使用
1. 打开在线链接
2. 创建项目（公司名称、项目名称、国家、平台、关键词等）
3. 点击项目进入任务管理页面
4. 点击"打开搜索"，在电商平台使用书签抓取数据
5. 回到监控工具，数据自动导入

### 团队使用
- 每个成员使用相同链接
- 数据各自独立存储
- 不会互相干扰
- 建议每个成员定期导出备份
