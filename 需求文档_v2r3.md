## 1. 筛选方式改为点击列标题（卡片模式）

### 1.1 设计模式
- **点击列标题弹出筛选卡片**
- 所有筛选卡片都有遮罩层背景（灰色：rgba(0,0,0,0.3)）
- 每个筛选卡片都有"确认"和"重置"两个按钮
- **删除列标题箭头**（▼/▲）

### 1.2 支持筛选/排序的列

#### 数值型列（售价、销量、侵权金额、评价数量）
**筛选卡片内容：**
1. **区间筛选**：从XX到XX（自己打字输入）
   - 售价：从100RMB到500RMB（根据国家显示货币符号）
   - 销量：从100到1000（无货币符号）
   - 侵权金额：从10RMB到50RMB（根据国家显示货币符号）
   - 评价数量：从100到500（无货币符号）
2. **数值筛选**：
   - `≥`（大于等于）
   - `从高到低`（降序）
   - `从低到高`（升序）
3. **确认和重置按钮**

#### 平台选择
**筛选卡片内容：**
1. **平台列表**：根据国家显示对应平台
   - 中国：淘宝、拼多多、京东、1688
   - 美国/欧盟/英国/韩国/其他：亚马逊、SHEIN、Temu、其他
2. **选择方式**：单选或多选
3. **确认和重置按钮**

#### 店铺名称
**筛选卡片内容：**
1. **排序选项**：
   - `升序`
   - `降序`
2. **筛选选项**：勾选或单选特定的店铺名
3. **确认和重置按钮**

#### 抓取时间
**筛选卡片内容：**
1. **日期选择**：日历选择起始时间和结束时间
2. **手动输入**：筛选的具体时间段手动输入
3. **排序选项**：
   - `升序`
   - `降序`
4. **确认和重置按钮**

#### 国家
**筛选卡片内容：**
1. **国家列表**：显示所有可用的国家
   - 中国、美国、欧盟、英国、韩国、其他
2. **选择方式**：多选
3. **确认和重置按钮**

### 1.3 不支持筛选的列（只显示内容）
- 序号
- 图片
- 侵权类型
- 侵权地址
- 链接

### 1.4 交互流程
```
用户点击列标题 → 显示遮罩层（灰色背景）→ 弹出筛选卡片 → 用户选择筛选条件 → 点击确认/重置 → 应用筛选/清除筛选 → 隐藏卡片和遮罩层
```

## 2. 表格横向滚动和列宽优化

### 2.1 列宽优化
1. **前面的列拉开距离**：序号、国家、图片、侵权金额、平台、侵权类型、商品名称、店铺名称、侵权地址
2. **从售价开始后面滚动**：售价、销量、评价数量、链接、抓取时间
3. **确保横向滚动功能正常工作**

### 2.2 文本截断+tooltip
以下列支持文本截断（最多显示2行，多的隐藏，鼠标移动到上面再显示完整内容）：
- **侵权地址**：最多显示2行，超出部分隐藏，鼠标悬停显示完整内容
- **商品名称**：最多显示2行，超出部分隐藏，鼠标悬停显示完整内容
- **店铺名称**：最多显示2行，超出部分隐藏，鼠标悬停显示完整内容

### 2.3 CSS要求
- 使用 `text-overflow: ellipsis` 或 `line-clamp` 实现2行截断
- 鼠标悬停时使用 `title` 属性或自定义tooltip显示完整内容
- 确保横向滚动功能：`.infringement-table-wrapper { overflow-x: auto; }`

## 3. 侵权金额显示优化

### 3.1 侵权金额计算
- 侵权金额 = 售价 × 销量
- 根据项目国家的货币符号显示
- 数值除以10000（万的单位）

### 3.2 侵权金额列标题
- 列标题显示货币单位（万RMB、万$等）
- 根据项目国家动态更新货币单位
- 不写"万"字，显示货币符号
- 例如：侵权金额（万RMB）、侵权金额（万$）

### 3.3 侵权金额显示逻辑
- 不显示"万"字，直接显示数字和货币符号
- 例如：125.36RMB（中国）、150.23$（美国）

## 7. 项目管理页面布局优化

### 7.1 需求说明
项目管理的上半部分（标题、按钮、搜索、筛选、平台）固定不动，不随页面滚动；
表头和表格数据一起滚动。

### 7.2 具体要求
1. **上半部分固定**：
   - 标题行（项目名称、已选数量）
   - 筛选区域（筛选项目下拉框 + 筛选公司下拉框）
   - 操作按钮行（四个操作按钮）
   - 平台选择区域（点击弹出卡片，根据国家动态显示）

2. **表格滚动区域**：
   - 表头和表格数据一起滚动
   - 支持横向滚动和纵向滚动
   - 表格宽度自适应
