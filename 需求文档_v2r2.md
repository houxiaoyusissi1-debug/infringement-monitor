## 1. 界面简化需求

### 1.1 筛选区域简化

**保留的元素：**
- 项目名字（侵权线索列表）
- 已选数量
- 筛选项目（下拉框）
- 筛选公司（下拉框）

**删除的元素（不要单独一行显示）：**
- ❌ 销量筛选（不筛选、≥、从高到低、从低到低 + 数量输入框）
- ❌ 评价筛选（不筛选、≥、从高到低、从低到低 + 数量输入框）
- ❌ 侵权金额筛选（不筛选、≥、从高到低、从低到低 + 万元输入框）
- ❌ 侵权类型筛选（复选框：商标侵权、专利侵权、版权侵权、其他）
- ❌ 抓取时间筛选（日期选择：开始日期、结束日期）

## 2. 筛选方式改为点击列标题

### 2.1 设计模式
- **点击列标题弹出筛选卡片**
- 列标题包括：侵权金额、销量、评价数量、售价等
- 点击后弹出卡片，显示筛选选项：
  - `不筛选`
  - `≥`（大于等于）
  - `从高到低`（降序）
  - `从低到高`（升序）

### 2.2 交互流程
```
用户点击列标题 → 弹出筛选卡片 → 用户选择筛选条件 → 自动应用筛选
```

## 4. 平台选择与国家关联

### 4.1 逻辑规则

**中国项目：**
- 显示平台复选框：淘宝、拼多多、京东、1688

**国外项目（美国/欧盟/英国/韩国/其他）：**
- 显示平台复选框：亚马逊、SHEIN、Temu、其他

### 4.2 动态显示
- 根据"筛选项目"中选定的项目的国家字段，动态切换平台复选框
- 项目数据结构包含 `country` 字段（中国/美国/欧盟/英国/韩国/其他）

## 5. 界面布局结构

### 5.1 顶部区域
```
第1行：项目名字 + 已选数量
第2行：筛选项目下拉框 + 筛选公司下拉框
第3行：四个操作按钮
第4行：选择平台（复选框，根据国家动态显示）
```

### 5.2 操作按钮
1. **+ 手动添加线索**（打开添加线索弹窗）
2. **🚀 打开搜索**（打开搜索页面）
3. **📥 导入抓取数据**（导入手动抓取的数据）
4. **📥 导出选中的侵权线索**（导出当前选中的线索）

### 5.3 表格区域
- 列数：14列
- 列标题可点击（触发排序/筛选）
- 表格支持横向滚动
